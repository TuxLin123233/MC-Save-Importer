# 使用指南

## 安装与运行

### 方法一：直接运行源代码

```bash
# 克隆仓库
git clone <仓库地址>
cd 存档管理器

# 创建虚拟环境（推荐）
python -m venv .venv

# 激活虚拟环境
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 运行程序
python src/main.py
```

### 方法二：使用可执行文件

从 [Releases](../../releases) 页面下载对应系统的可执行文件，直接运行即可。

---

## 首次使用

### 1. 选择 .minecraft 文件夹

首次导入存档时，程序会询问你的 `.minecraft` 文件夹位置：

**常见位置：**
- **Windows**: `%APPDATA%\.minecraft`
- **Linux**: `~/.minecraft`
- **macOS**: `~/Library/Application Support/minecraft`

### 2. 选择 ZIP 文件夹

选择存放地图 ZIP 文件的文件夹，程序会自动识别其中的所有 ZIP 文件。

---

## 导入存档

### 标准结构

如果你的 Minecraft 使用标准结构（存档在 `.minecraft/saves`）：

1. 点击「导入存档」按钮
2. 选择 ZIP 文件夹
3. 等待导入完成

### 版本迁移结构

如果你的 Minecraft 使用版本迁移结构（存档在 `.minecraft/versions/<版本>/saves`）：

1. 点击「导入存档」按钮
2. 选择要导入到的游戏版本
3. 选择 ZIP 文件夹
4. 等待导入完成

### 重复存档处理

如果目标位置已存在同名存档，程序会提示：
- **覆盖**：删除旧存档，导入新存档
- **跳过**：保留旧存档，跳过该 ZIP 文件

---

## 配置文件

程序会在项目根目录创建 `data.json` 文件存储配置：

```json
{
  "minecraft_path": "/path/to/.minecraft",
  "migrate": false
}
```

- `minecraft_path`: `.minecraft` 文件夹路径
- `migrate`: 是否为版本迁移结构

如需重新设置路径，直接删除此文件即可。
